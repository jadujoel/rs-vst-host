Perhaps we should allocate an arena for the plugins to use for their allocations, and then we can free the entire arena when the plugin is unloaded. This would simplify memory management for the plugins and ensure that we don't have any memory leaks when they are unloaded. And we can make the arena thread-local, so that each plugin has its own arena to work with. This way, we can avoid any potential issues with concurrent access to the arena from multiple plugins.
